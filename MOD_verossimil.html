First things first

  1. Crie um diretório de trabalho na raiz do seu computador. Evite muitos subdiretórios;
  2. Por favor, instale o programa IQ-TREE se quiser trabalhar com partições diferentes.
  3. Se não for o seu caso, use o IQ-TREE web server.

  • Na seção Tree Inference, coloque os dados no campo Alignment file: da subseção Input Data, conforme exemplos fornecidos pelo próprio sítio web.

  • No campo Substitution model: da subseção Substitution Model Options, escolha o tipo de modelo mais verossímil aos seus dados. Lembre-se que para partições é melhor usar o programa baixado em seu computador.

  • No campo Rate heterogeneity:, você pode optar por usar o parâmetro Gamma[+G], caso os caracteres de sua matriz não evoluam numa mnesma taxa! Se as taxas forem diferentes, porque nem todos os os caracteres evoluem na mesma taxa (FITCH & MARGOLIASH, 1967), assumimos que as taxas variem ao longo dos diferentes caracteres de acordo com uma distribuição gama (YANG, 1993) discretizada em, no mínimo, quatro categorias. Podemos, inclusive, escolher o número de categorias no campo #rate categories:. Se a frequência de unm estado de caráter for alta, observaremos mais transições de um estado de caráter para esse estado do que o contrário. Esse modelo é denominado among-character rate variation (ACRV model). O parâmetro +I refere-se à presença de caracteres que não variem, o que é praticamente impossível de acontecer, principalmente quando usamos matrizes de caracteres baseados em morfologia. Dessa forma, o campo Invar. sites[+I] geralmente não é marcado. Cuidado com esse parâmetro: coisas ruins acontecem quando abrimos as portas do inferno!

  • O campo FreeRate heterogeneity: faz a distribuição gama variar ao longo da análise.

  • O campo Ascertainment bias correction: corrige o problema de caracteres não informativos como autapomorfias ou sítios que não mudam entre as espécies da matriz. Esse é mais um parâmetro de um modelo hierárquico que estamos criando: o parâmetro +ASC.

  • Na subseção Branch Support Analysis, vamos ajustar coisas para obtermos suportes dos ramos com uma análise de bootstrap não paramétrico, conforme visto em sala de aula. No campo Bootstrap analysis:, tomem cuidado com a opção ultrafast porque ela pode produzir valores sem sentido para um ramo, tendo em vista que nesse tipo de procedimento não há reamostragem de toda a matriz, mas leva em conta valores de algumas pseudoréplicas já estimadas. Na opção standard, só podem ser realizadas 1000 reamostragem com substituição.

  • Na subseção IQ-TREE Search Parameters, a opção Pertubation strength: permite dizer que proporção de ramos das árvores iniciais produzidas (starting trees) será usada para gerar pertubações.

  • A opção IQ-TREE stopping rule: é um limite que colocamos para que, durante a busca, o procedimento ainda fique durante n vezes tentando achar árvores mais verossímeis. Ao final de n vezes, se a busca falhar em achar árvores mais verossímeis, uma nova rodada inicia-se.
